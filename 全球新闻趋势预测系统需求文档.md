# 全球新闻阅读影响趋势预测系统需求文档

## 1. 引言

### 1.1 背景与目标

本系统旨在提供一个融合**反身性理论**（索罗斯）、**证伪主义**（波普尔）和**反脆弱**（塔勒布）理念的金融新闻分析与预测推演训练平台。用户通过阅读原始新闻、提取事实、做出可验证的概率推演，并与大语言模型（LLM）的预测进行对比，从而培养“承认无知、保持灵活、控制下行风险”的决策能力。

系统的核心目标包括：

* 养成“先看事实，再下判断”的习惯，通过制度对抗认知偏差（行为金融学视角）。
* 强制设定**可证伪的判定规则**和**预期兑现日**，使预测有边界、可验证、可复盘。
* 通过**虚拟仓位管理**、**不对称比**、**反脆弱气泡图**等工具，刻意练习“错时亏小钱、对时赚大钱”的反脆弱下注能力。
* 提供人类与 LLM 预测的盲测对抗，形成客观的自我纠错闭环。

### 1.2 范围

系统覆盖新闻源采集、噪声动态过滤、人类预测推演提交、LLM 对照组生成（含降本熔断控制）、自动化判定状态机、冷热数据生命周期管理、反脆弱统计可视化及行为偏误强制复盘等模块。**本系统不包含任何真实资金交易功能，所有仓位及资产均为虚拟训练用途。**

* 《Study of Financial Economics》—— 学习框架与哲学根基

## 2. 总体描述

### 2.1 产品视角

系统采用“盲测对抗、事后验证”的产品逻辑。系统只负责采集和呈现原始新闻，**严禁引入任何前置的 AI 摘要或 AI 情感标注**，必须由人类用户亲自阅读全文、提取事实并下注。下注后，后台异步生成 LLM 对照预测。最终由判定器依据客观数据对双方进行同步裁决。

### 2.2 核心业务规则总览 (Business Rules)

为确保系统设计的严谨性，特设定以下库级与业务级核心硬约束：

* **[BR-01 虚拟仓位强校验规则]**：虚拟仓位（`position_size_pct`）是训练风险管理与反脆弱的核心指标。**该字段在业务层、前端表单验证、数据库约束中均为强制必填项（NOT NULL）**，禁止默认值或空值。
* **[BR-02 LLM 对照组熔断与降本规则]**：系统必须内置 LLM 调用熔断器。当新闻被判定为高频噪声、文本过短、或发布时间已成明日黄花时，系统应主动放弃（Skip）调用大模型对照组，标记 `llm_skip=true`，以节约 API 算力成本，并防止大模型因缺乏上下文产生严重幻觉。
* **[BR-03 噪声自适应反馈闭环规则]**：噪声标记不应仅停留在前端隐藏，必须反哺采集端。当某条新闻在 7 天内被独立专家用户标记达到 3 次（或管理员一键确认）时，系统需动态提取其高频特征词，反向注入采集模块的动态排除词库。
* **[BR-04 到期自动判负规则]**：证伪主义的核心在于“过时未证实即为伪”。当系统时间越过预期兑现日（`target_date`）且关联的外部自动化判定规则未被触发时，**状态机必须通过原子事务一键将人类与 LLM 的预测结果（outcome）强制判定为 0（证伪）并关闭流转**，拒绝数据处于永久挂起的中间态。
* **[BR-05 行为偏误强制复盘规则]**：针对发生严重认知偏差的推演场景（高置信度错误、低概率命中），系统将锁定其状态并红标警告，**强制要求用户填写反思复盘文本（`post_mortem`）**，否则不予解除统计面板的警示。

## 3. 功能需求

### 3.1 新闻采集与自适应噪声过滤

#### 3.1.1 动态源轮询

* 系统必须支持管理员动态增删改查 RSS/Atom 新闻源。
* 采集调度器需根据各新闻源配置的刷新间隔（`refresh_interval_sec`）进行异步轮询，并支持配置参数热加载。

#### 3.1.2 库级去重防御

* 必须基于原始新闻的 `URL` 实施全局唯一性去重；若无有效 URL，则通过 `标题 + 发布时间` 生成的哈希值实施唯一性拦截，严防信息流重复刷屏。

#### 3.1.3 前置过滤与噪声闭环

* **静态过滤**：新闻正文或标题必须包含 `require_keywords` 且不含 `exclude_keywords` 才能入库。
* **动态反哺（依据 BR-03）**：系统开放用户端“🗑️ 标记为噪音”功能。触发阈值后（7天内独立用户标记达3次或管理员审阅），系统需将高频噪声特征词异步加载至采集器的排除词库中，实现采集器的自适应噪声进化。

### 3.2 预测推演与 LLM 熔断下注

#### 3.2.1 推演表单硬约束

用户进入新闻详情页阅读全文后，点击展开推演决策区。表单包含以下字段及约束：

* **摘取事实 (可选)**：鼓励用户提炼客观事实。
* **逻辑推演 (必填)**：详细描述推演逻辑。
* **主观胜率 (必填)**：0-100% 滑块。
* **虚拟仓位比例 (强制必填，依据 BR-01)**：0-100% 滑块，不允许为空。
* **预期兑现日 (强制必填，依据 BR-04)**：日期选择器，最小可选日期为 `当前日期 + 1天`。
* **自动化判定规则 (必填)**：用户必须选择并配置至少一个判定模板（如价格涨跌、央行决议等）。

#### 3.2.2 LLM 盲测组生成与熔断（依据 BR-02）

人类提交预测后，触发异步队列。生成 LLM 预测前，系统必须依次检查**熔断检查流水线**：

1. 检查当前新闻是否已在 `noise_flags` 中被标记。
2. 检查新闻正文字符数是否小于 100 字。
3. 检查当前时间距离新闻发布时间是否超过 3 天。

* **满足以上任一条件，或大模型 API 返回弃权时，系统自动拦截调用，将关联的 `llm_skip` 设为 `true**`。
* 若通过熔断检查，则调用大模型生成对照推演，并建立 `parent_prediction_id` 关联。

### 3.3 自动化判定与状态机控制

#### 3.3.1 状态机流转定义

预测数据的判定状态字（`judge_status`）必须遵循以下标准状态机流转：

* `pending`：初始状态。人类提交后、到达 `target_date` 之前。
* `judging`：正在由判定引擎连接外部 API 进行规则解析中。
* `resolved`：判定成功，结果（`outcome=0` 或 `1`）已成功锁死并写入。
* `failed_api`：因网络故障或外部接口失效导致的异常挂起态，提供管理员手动修正入口。

#### 3.3.2 定时验证与到期自动判负（依据 BR-04）

* 系统每小时扫描未决预测，解析 `rule_json` 并调用对应判定器（如 Yahoo Finance API 验证股价、特定经济数据接口验证 CPI 等）。
* **到期强制终止**：若扫描时发现系统当前时间已超过 `target_date`，且规则仍未触发，判定器必须执行**同步判负事务**：强制将该人类预测及对应的 LLM 预测的 `outcome` 置为 0，状态流转至 `resolved`，并将 `verified_at` 设为当前时间。

### 3.4 统计、反脆弱可视化与强制复盘

#### 3.4.1 Brier 对抗曲线

* 每日定时计算人类与对照组 LLM 的累计 Brier Score。
* 前端提供日历史快照的对比曲线，负数差值（人类分数 - LLM 分数）代表人类认知击败 AI 的程度。

#### 3.4.2 反脆弱核心指标——不对称比 (Asymmetry Ratio)

* 系统需实时统计用户在置信度 $\ge 50\%$ 的预测中：`正确预测时的平均虚拟仓位 / 错误预测时的平均虚拟仓位`。
* 系统设定指标**达标线为 1.5**。若低于 1.5，前端卡片必须显示黄色脆弱性警告。

#### 3.4.3 反脆弱气泡图与虚拟净值曲线

* **反脆弱气泡图**：X 轴为胜率，Y 轴为实际结果（0 或 1），气泡半径为仓位比例。直观标红“重仓大错”的脆弱点。
* **虚拟净值曲线**：同时绘制两条虚拟资产收益曲线——“等权重净值曲线”与“仓位加权净值曲线”，用于直观验证仓位管理对最终业绩的脆弱/反脆弱重塑效应。

#### 3.4.4 行为偏误强制复盘黑名单（依据 BR-05）

系统必须自动捕获以下两类产生严重确认偏误的推演记录，并锁入“待复盘黑名单”：

1. **过度自信（高置信度错误）**：用户主观概率 $\ge 70\%$ 且实际结果被证伪（`outcome=0`）。
2. **幸存者偏差（低概率命中）**：用户主观概率 $\le 30\%$ 且实际结果被证实（`outcome=1`）。

* **拦截约束**：黑名单不为空时，前端首页和统计页需展示悬浮红色通知。**用户必须在表单中提交 `post_mortem` 反思文本后，才能解除红标挂起状态。**

## 4. 非功能性需求

### 4.1 性能与并发

* 采集 Worker 必须采用独立的轻量级异步线程池（如 Tokio task），不得阻塞主 Web 服务的 API 响应。
* 自动化判定扫描应采用分批（Batching）锁机制，防止单次扫描大批量历史数据时导致数据库产生死锁或长连接阻塞。

### 4.2 安全与权限

* 系统分为 `human_expert`（推演专家）与 `admin`（系统管理员）两种角色。
* 只有管理员具备增删采集源、一键注入高频噪声词、以及对状态为 `failed_api` 的判定记录进行手动 outcome 修正的权限。

### 4.3 数据留存与冷热分离策略

* **冷数据定义**：系统将发布时间超过 7 天（`published_at < CURRENT_DATE - 7`）且 **无任何人类或 LLM 预测关联** 的历史新闻定义为冷数据。
* **物理切除归档策略**：为了将数据库的存储压力控制在常数级别，系统必须挂载常驻后台 Worker，在每天凌晨 3:00 自动执行清理事务。
* 清理事务必须通过 `NOT EXISTS` 安全校验，**彻底清空（UPDATE 设为 NULL）无预测冷新闻的 `content`（正文全文）字段**。数据库仅保留其 `title` 和 `url` 作去重索引依托。冷数据正文切除后不可逆，前端将关闭这些冷新闻的详情页入口。

## 5. 名词解释

| 术语 / 指标 | 定义与业务含义 |
| --- | --- |
| **Brier Score** | 概率预测准确度的客观量化指标。得分越接近 0 准确度越高，越接近 1 准确度越低。 |
| **不对称比 (ASYM)** | 确信时重仓、不确定时轻仓的量化指标。公式：正确平均仓位 / 错误平均仓位，用于评估下注策略是否具备反脆弱性。 |
| **可证伪判定规则** | 包含具体参数和可获取边界的可执行规则体（JSON 格式），如“在指定日期苹果股价是否大于 180 美元”。 |
| **冷热分离** | 区分高频决策数据（热数据）与纯去重索引数据（冷数据）的运维策略，过期无预测新闻的正文将被物理清除。 |
| **llm_skip (大模型熔断)** | 判定是否跳过大模型对照生成的控制开关，用于低质新闻、噪声新闻及严重过期新闻的降本拦截。 |

| 哲学理念 | 核心功能点与业务规则映射 | 验收通过标准 |
| --- | --- | --- |
| **证伪主义** (波普尔) | 每条预测绑定 `target_date` 与 `rule_json`；<br>**依据 BR-04 逾期未决自动判负。** | 到期未决的数据不会永久挂起，系统会在到期日凌晨或扫描周期自动结算 outcome=0。 |
